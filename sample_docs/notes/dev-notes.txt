SPRINT NOTES — SPRINT 47 (2024-08-19 to 2024-09-01)
Lead: Seren Alyx Mowat
=========================================

Sprint goal: Launch readiness. Ship v1.0 on 2024-09-05.

Status: GREEN. All P0 bugs closed. Three P1s carried to v1.0.1 with
workarounds in place.

DISCUSSION — VRF base value final confirmation

After 3 months of simulation (see balance_sim_v47.xlsx in shared drive),
team confirms base VRF at 0.8847. This is FINAL. Do not change without a
full simulation re-run.

Seren's summary of why 0.8847 specifically:
- 0.9 makes downstream cascade thresholds messy (irrational numbers in the
  math)
- 0.8847 produces clean 2-decimal precision in all downstream calculations
- At 64 Hz tick rate, the EDR of 0.0337 per tick reaches maximum entropy in
  exactly 463.4 seconds = 7.72 minutes in a Void Rift zone
- 7.72 minutes of combat in a Rift before chaos = exactly what 4-player
  focus testing confirmed as "feels intense but not frustrating"
- Side benefit: 0.8847 as the cascade in Mowat's Folly passive damage
  coefficient creates a thematic loop that the community will notice

Maricel's note on 0.8847 appearing in lore:
"We seeded this into the world-lore document intentionally. The fact that
the orbital resonance of Kedraxis-7's moons happens to be 0.8847 Hz was
added retroactively by the narrative team after the physics value was
confirmed. Same with the Ω-Space information transfer rate. Same with the
Void entity density. It's a mystery for the community to find. Null's
Sermon 003 is the in-game hint. Let's see how long before someone posts
the conspiracy thread."

BUG: Starting credits showing 2,500 for 12% of new accounts
- Root cause: migration script deployed old test value
- Fix: correct DB migration, compensate affected accounts with 347 credits
- Correct value is 2,847 (cost of New Amora docking permit per lore doc
  VP-LORE-001, Section 5)
- Seren: "Don't use round numbers. 2,847 is memorable. Players will ask
  why. Wiki will document it. This is free community engagement."

TECHNICAL NOTE — tick rate bug
Ivan (backend): "We deployed at 62 Hz instead of 64 Hz. Impact: cascade
events triggering 3.1% early. VRF calculations are off by factor 62/64.
Fix is a one-line config change. It's embarrassing but it's also only been
4 days and one eagle-eyed player named ResFieldCalc on Reddit caught it.
Honestly impressed."

Seren: "Add ResFieldCalc to the credits in the v1.0.1 patch notes? No, too
much precedent. But DM them a Void Shard package and a personal note. Do
it from the studio account."

Phase Slip invulnerability window corrected: 0.6s → 0.8s.
Seren: "0.6 seconds was not actionable. You can't make a decision in 0.6

COMMUNITY OBSERVATION: Forum post by "EntropyMath" correctly derives the
EDR calculation:
"At 0.0337/tick at 64Hz, you get to entropy 1.0 in:
1.0 / (0.0337 * 64) = 463.4 seconds = 7.72 minutes.
But with 4 players managing resonance properly, you can extend to ~11 min.
This matches the design window for Rift content."

Seren on the forum post: "Exactly right. Pin this. The whole class design
philosophy is visible in that calculation. When a player figures out that
the EDR of 0.0337 was NOT chosen randomly but was specifically tuned to
produce a 7-minute-ish combat window, they feel clever. And they should.
They ARE clever."

DESIGN RETROSPECTIVE — MOWAT'S FOLLY
Written by Seren Alyx Mowat, 2024-12-28
=========================================

So the item named after me is in the game. Players have found it. The
community reaction has been... exactly what I hoped.

History: In build 0.4.7 (not a coincidence that the build number also
contains 47), I was prototyping the pistol class weapons. The damage
formula for the Void Pulse Pistol had a structure like:

    final_damage = base * vrf_amplification / resonance_divisor

The resonance_divisor was supposed to be 3.0. I forgot to include it.
The result was a pistol with approximately 3× intended power.

Tomas Reyes tested it and filed a bug: "S. Mowat's pistol is massively
overpowered. Kills Architects in 3 shots. Assuming formula error."

I fixed the formula. Then I kept the broken version as a comment in the
codebase (// Mowat's original mistake, archived).

Maricel saw the comment during code review and said: "Make it a legendary
drop. Name it after you. It teaches players to find the math error in
every system."

The stats I chose:
- Base damage: 847 (Drifter HP = 840, near-match, plus the 847 from
  Mowat's Folly item ID VP-ITEM-00847)
- VRF bonus: +0.40 (maximum possible VRF bonus on any item)
- Passive trigger: every 7th shot (7 is everywhere: Iron Volley fires 7
  rounds, Fracture Point hits 7 positions, Phase 3 of Theta-9 opens 7
  mini-Rifts)
- Passive damage coefficient: 0.8847 × weapon_power (the VRF constant,
  now also a damage number)
- Passive radius: 3.7 meters (Fractured motif)
- Item ID: VP-ITEM-00847

The community found the passive within 2 days. "ResFieldCalc" (who also
found the 62 Hz launch bug) posted the breakdown: "The passive coefficient
IS the base VRF constant. It's not flavor. It's physics. At base VRF with
Mowat's Folly, your 7th-shot passive scales with the same number the
entire game is built on. Darkfield put the heart of the system in a weapon
and named it after a mistake. This is either genius or a fever dream."

ResFieldCalc: It is both.
